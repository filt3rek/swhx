This tutorial show how to handle the FSCommand events from an ActionScript or haXe application. In this example we handle the "save_file" fscommand and use it to save some data on the local drive.

Compilation

To compile, open up terminal or the command prompt, navigate to the sample folder and issue:

haxe compile.hxml

Execution

To run the sample, issue:

neko app.n

Alternatively, you can launch with the Screenweaver HX boot loader, found in <$haxlib path swhx>/tools. Doing so requires all build materials to be in the same folder as the boot loader.

*NOTE* - this sample expects MTASC to be installed to your system.
